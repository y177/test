안녕하세요! 저는 Claude Code로, 다양한 소프트웨어 개발 작업을 도와드릴 수 있습니다. SuperClaude 프레임워크를 통해 제공되는 주요 기능들을 소개해드리겠습니다:

🛠️ 개발 & 구현
/build - 프로젝트 빌드, 프레임워크 자동 감지
/implement - 기능 구현, 컴포넌트 생성
/design - 시스템 아키텍처, API 설계
🔍 분석 & 조사
/analyze - 코드베이스 분석, 성능/보안 검토
/troubleshoot - 문제 진단 및 해결
/explain - 코드/개념 설명
✨ 품질 & 개선
/improve - 코드 품질 향상, 최적화
/cleanup - 기술 부채 제거, 리팩토링
/test - 테스트 실행 및 생성
📝 문서화 & 버전관리
/document - 문서 생성
/git - Git 워크플로우 지원 (커밋, PR 생성)
🎯 프로젝트 관리
/task - 장기 프로젝트 관리
/spawn - 복잡한 작업 분산 처리
/estimate - 개발 추정
🚀 특별 기능
MCP 서버 통합:

--c7 Context7 (라이브러리 문서)
--seq Sequential (복잡한 분석)
--play Playwright (브라우저 자동화)
--ultra-mcp 멀티 AI 모델 통합
전문 Persona:

--persona-frontend UI/UX 전문
--persona-backend 서버/API 전문
--persona-security 보안 전문
--persona-architect 아키텍처 전문
그 외 7개의 전문 페르소나
구체적으로 어떤 작업을 도와드릴까요?

위의 /task 나 /documet 는 .claude 폴더에 command 폴더에 따로 md 파일이 있어야 명령이 작동하는데..

맞습니다! 제가 설명드린 명령어들은 SuperClaude 프레임워크의 사용자 정의 슬래시 커맨드입니다.

현재 프로젝트에서 실제로 사용 가능한 슬래시 커맨드는 다음과 같습니다:

📋 사용 가능한 커맨드
/sc:analyze      - 코드 품질, 보안, 성능, 아키텍처 분석
/sc:build        - 빌드, 컴파일, 패키징
/sc:cleanup      - 코드 정리, 데드 코드 제거
/sc:design       - 시스템 아키텍처, API 설계
/sc:document     - 컴포넌트/기능별 문서화
/sc:estimate     - 개발 추정
/sc:explain      - 코드/개념 설명
/sc:git          - Git 작업 및 커밋 메시지
/sc:implement    - 기능 및 코드 구현
/sc:improve      - 코드 품질 및 성능 개선
/sc:index        - 프로젝트 문서화 및 지식 베이스
/sc:load         - 프로젝트 컨텍스트 분석
/sc:spawn        - 복잡한 작업 분산 처리
/sc:task         - 복잡한 작업 워크플로우 관리
/sc:test         - 테스트 실행 및 리포트
/sc:troubleshoot - 문제 진단 및 해결
/sc:workflow     - PRD로부터 구조화된 워크플로우 생성

물론 슬래시 커맨드 없이도 직접 요청하시면 동일한 작업을 수행할 수 있습니다. 예를 들어:

"코드 분석해줘"
"이 기능 구현해줘"
"문서 작성해줘"
어떤 작업을 도와드릴까요?
